Resampling Methods

 

 

Abstract:

 

The resampling techiques are into mainstream as they are much required for data analysis. The goal of resampling methods which are used in statistics is to make use of the existing training data and make correct predictions about the performance of the algorithm on test data which is the unseen data. There are two ways to choose a particular algorithm over other algorithms,first is to evaluate the algorithm on training data and the other is to use statistical methods called resampling methods that allow you to predict the performance of your algorithm over test data..

 

1:Introduction:

 

The Resampling is a term which does not apply only to machine learning but it applies to all the fields where you want to obtain precision parameters about the data like (mean,median and mode) to name a few .The meaning of resampling itself is to perform the process of sampling repeatedly. In this article we will focus on the principles of resampling and various resampling methods possible. Resampling is a method of constantly drawing samples from the collection of actual dataset and it is a non parametric method of inferences. Resampling involves selection of data from the original data .So due to selection of the data with replacement from the original data the sampled data can be repetitive.There is no predefined size for taking the samples in the method of resampling.The larger the size of sample,the more the accuracy of the algorithm that will be using that sample.

There can be an issue of overfitting in case of sampled data but that can be solved by combination of one method with other.Resampling provides more useful samplesets which can play a major role while training process.These are the different types of resampling methods:

 

1.1: Training& Test Split:

This is the most simples resampling method .The main points regarding the method are:

A)    Division of dataset into two parts: a) Training data b) Test data

B)    Random rows from datasets is selected and labelled as training data and rest as test dataset

C)    Generally the split between training and test data is 60:40

 

Advantages:

 

A)    This technique is suitable for large datasets

B)    This technique is very fast.

 

Disadvantages:

 

A)    This technique lead to high variance

B)    Less accurate

 

1.2:K-fold cross validation:

In this method the noise which is a limitation in case of first method is removed.This method provides a better evaluation of the algorithm.The main points regarding the method are:

A)    Division of the datasets into k groups.

B)    Training of the individual k groups for the algorithm.

C)    Each group of data is called fold ,so the name k-fold

D)    Training of the algorithm is done on k-1 data first and one data is held out which work as test data. Same procedure is applied for all the rest k-1 pints so that equal opportunity is given to all the data points to act as test data.

Advantages:

 

A)    This technique is applicable to algorithms with less variance

B)    More accurate

C)    The model buid using cross validation does not overfit.

 

Disadvantages:

 

A)    More time consuming

B)    Complex to understand

C)    Bias-variance trade off associated with choice of k.

This technique help ensure data is not overfit.

 

1.3:Leave one out Cross validation(LOOCV):

When cross validation is done in such a way that the value of k is taken to be 1 it is

known as Leave one out cross valdation(LOOCV)..

Advantages:

A)    This technique is simple.

B)    It does not suffer from exhaustive calculations

 

Disadvantages:

A)    As we test against only one data point our computation gets highly influenced

B)    As we repeat cross validation n times(where n is the number of data points) this can lead to higher execution time.

 

1.4:Repeated Random Test-Train:

In this technique we used to randomly split the data and repeat the same work of splitting and testing datapoints repeatedly randomly.This method generally uses 66%33% (testsplit.

 

Advantages:

A)    Less variance

B)    Repeat the experiment as per your requirement. No upper limit set.

 

Disadvantages:

A)    When evaluating the algorithm with this method, it may involve much the same data in each execution of training/testing.

B)    This technique can be very exhaustive at times.

 

1.5:Bootstrapping

This is another method of resampling in machine learning. The main points of this method are:

A)    Divide the dataset (size N) into training data and bootstrap data

B)    The division of data should be in such a way that bootstrap samples should also be of size N

C)    Bootsrap samples serve as test data

D)    Run your model on each bootstrap sample and evaluate the performance by measuring the statistics associated with model.

Advantages:

A)    This technique reduces sampling error.

B)    It does not require large sample size.

Disadvantages:

A)    It does not work well for small sample sizes

B)    It is a time consuming process

 

1.6:Jackniffing:

This is another method of resampling in machine learning.The main points of this method are:

A)    Divide the dataset (size N) into bootstrap data of size (n-1) which act as test data.

B)    The division of data should be in such a way that bootstrap samples should be of size N-1

C)    Bootsrap samples serve as test data

D)    Run your model on each bootstrap sample and evaluate the performance by measuring the statistics associated with model.

Advantages:

A)    This technique require less computation as compared to Bootstrapping

B)    This technique has reduced bias

Disadvantages:

A)    This method fails for non smooth statistics which Bootstrapping method does not suffer from.

 

2. History/Evolution of the approach with related work:

 

The history of resampling methods date back to 1935 when R.A.Fisher has introduced the concept of randomization in his experiments at Rothamsted Experimental station in which he demonstrated with his famous Experiment i.e A Lady Tasting Tea which is stated as under:.

A lady Tasting Tea: A lady discovers that by tasting a cup of tea made with milk she can distinguish whether milk was added frist or tea was added first while preparing a cup of tea.

For the experiment to take place 8 cups of tea were placed and it was decided to classify into two sets A & B which mean Milk first and Tea first respectively.From this experiment we want to find

A)    the difference between A & B.

B)    standard error of their difference

C)    perform hypothesis test to see whether A & B differ

Similarly,In 1948 cross validation which we was one of the resampling method is proposed by Krutz which is stated as under:

 

Divide the set z into two sets , one is testing set and other is training set.The parameter α is estimated on the training data set.The cross validation is the prediction error calculated using the test sample.

Models are generated for both the sub-samples and then both the samples are used to generate the cross validations.

The original objective of cross validation is to verify replicability of results.

3. Algorithm with mathematical notation

 

Algorithm for Train/Test split:

 

Step 1: Randomly assign data into two groups

Step 2: For k=1,.,k estimate wλK on all the training blocks.

Step 3:Compute the error on testing block

Step 4:Compute the average error CV(λ)=1/k∑error(λ)k

Repeat the procedure for each choice of λ.

 

Algorithm for k-fold cross validation:

 

Step 1: Randomly assign data into k groups(use same division of data for all other steps)

Step 2: For k=1,.,k estimate wλK on all rest training blocks.

Step 3: Compute the error on testing block i.e. errork(λ)

Step 4: Compute the average error CV(λ)=

Repeat procedure for each choice of λ

 

Algorithm for Repeated Random cross validation:

 

Step 1: Randomly assign data into 2 groups namely training data and testing data.

Step 2: For each k training group the model is fit to the training data and tested on the testing data.

Step 3:The above procedure is repeated randomly taking some as training and other as testing data.

Step 3:Compute the error on testing block i.e. errork(λ)

Step 4: Compute the average error CV(λ)=

Repeat procedure for each choice of λ

 

Algorithm for Leave One Out Cross Validation:

 

Step 1: Randomly assign data into 2 groups

Step 2: Division of training and testing block should be such that if the length of training block is n-1 then the length of testing block is one..

Step 3: Repeat the same process of training and testing such that testing block takes exactly one of the element from the training block and rest of it forms the training block.

Step 3: Compute the error on testing block i.e. errork(λ)

Step 4: Compute the average error CV(λ)=

Repeat procedure for each choice of λ

 

Algorithm for Bootstrapping:

 

Step 1: Randomly assign data into 2 groups namely training and testing data which is bootstrap samples in this case.

Step 2: Construction of Bootstrap samples should be done in such a way that with replacement the sample needs to be prepared which is also of the same size as original data.

Step 3: Repeat the same process of training and testing on k testing blocks or bootstrap samples.

Step 3: Compute the error on testing block i.e. errork(λ)

Step 4: Compute the average error CV(λ)=

Repeat procedure for each choice of λ

 

Algorithm for JackKniffing:

 

Step 1: Randomly assign data into 2 groups namely training and testing data which is bootstrap samples in this case.

Step 2: Construction of Bootstrap samples should be done in such a way that with replacement the sample needs to be prepared which is also of the size n-1.

Step 3: Repeat the same process of training and testing on k testing blocks or bootstrap samples.

Step 3: Compute the error on testing block i.e. errork(λ)

Step 4: Compute the average error CV(λ)=

Repeat procedure for each choice of λ

 

4. Example Problem with Solution:

 

The problem is to validate nave bayes classification algorithm using k-fold cross validation.The code for the same is given below:

 

import nltk

import numpy as np

from sklearn.model_selection import KFold

kf=KFold(n_splits=3)

sum=0

for train,test in kf.split(data):

train_data=np.array(data)[train]

test_data=np.array(data)[test]

classifier=nltk.NaiveBayesClassifier.train(train_data)

sum+=nltk.classify.accuracy(classifier,test_data)

average=sum/3

 

The example above show implementation of k-fold cross validation on Nave Bayes classification algorithm.In this example what we have done is:

1. We set our K parameter to be 3

2. We use split function which will return three folds of data containing two arrays-one needed for training and another needed for testing

3. We extract only those examples from our original dataset with the required indices for every single fold.

import pandas

from sklearn import model_selection

from sklearn.linear_model import LogisticRegression

url = "https://archive.ics.uci.edu/ml/machine-learning-databases/pima-indians-diabetes/pima-indians-diabetes.data"

names = ['preg', 'plas', 'pres', 'skin', 'test', 'mass', 'pedi', 'age', 'class']

dataframe = pandas.read_csv(url, names=names)

array = dataframe.values

X = array[:,0:8]

Y = array[:,8]

test_size = 0.33

seed = 7

X_train, X_test, Y_train, Y_test = model_selection.train_test_split(X, Y, test_size=test_size, random_state=seed)

model = LogisticRegression()

model.fit(X_train, Y_train)

result = model.score(X_test, Y_test)

print("Accuracy: %.3f%%" % (result*100.0))

import pandas

from sklearn import model_selection

array = dataframe.values

X = array[:,0:8]

Y = array[:,8]

num_instances = len(X)

seed = 7

kfold = model_selection.KFold(n_splits=10, random_state=seed)

model = LogisticRegression()

results = model_selection.cross_val_score(model, X, Y, cv=kfold)

print("Accuracy: %.3f%% (%.3f%%)" % (results.mean()*100.0, results.std()*100.0))
